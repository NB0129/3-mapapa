■ 牌の定義（MahjongLogic.gd）
--------------------------------------------------
  【牌ID】
    萬子: MAN_1=11, MAN_9=19 （1萬と9萬のみ）
    筒子: PIN_1〜PIN_9 = 21〜29
    索子: SOU_1〜SOU_9 = 31〜39
    字牌: EAST=41, SOUTH=42, WEST=43, NORTH=44, HAKU=45, HATSU=46, CHUN=47

  【牌の属性フラグ】
    is_red: bool        → 赤ドラ（5筒×2・5索×2・北×1）
    is_gold: bool       → 金ドラ（8筒×1・8索×1）
    is_haku_pochi: bool → 白ポッチ（白4枚のうち1枚）

  【デッキ構成（108枚）】
    1萬 4枚 / 9萬 4枚
    1〜9筒 各4枚（計36枚）　5筒に赤2枚・8筒に金1枚
    1〜9索 各4枚（計36枚）　5索に赤2枚・8索に金1枚
    東・南・西 各4枚（計12枚）
    北 4枚（うち赤北1枚）
    白 4枚（うち白ポッチ1枚・通常白3枚）
    發・中 各4枚（計8枚）

■ 山・王牌
--------------------------------------------------
  - シャッフル後に末尾8枚を嶺上（rinshan）として分離
  - 嶺上の先頭をドラ表示牌（dora_indicators[0]）
  - その次を裏ドラ用（ura_dora_indicators[0]）
  - 残りがツモ山（wall）
  - カン時: wallの先頭をドラ表示牌として追加 → wallが2枚減少
  - 北抜き時: rinshanから補充 → wallに影響なし

■ ゲーム進行ルール
--------------------------------------------------
  戦型      : 東南戦（西入なし）
  開始点数  : 35000点
  チー      : なし
  ダブロン  : なし（頭ハネ。捨てたプレイヤーから次順に近い和了者を優先）
  連荘      : テンパイ連荘
  オーラス  : あがり止めあり
  西入      : なし（南場オーラス後に終局）
  飛び      : 0点以下で即終局
  積み棒    : 1本場 = 2000点増し（ツモ時は1000点×2人）
  北        : 花牌扱い（河に捨てられない・抜いて嶺上から補充）

  【流局テンパイ精算（4000点システム）】
    テンパイ2人・ノーテン1人: ノーテン者が4000点払い・テンパイ2人に2000ずつ
    テンパイ1人・ノーテン2人: ノーテン2人が2000ずつ払い・テンパイ1人が4000受取
    全員テンパイ or 全員ノーテン: 支払いなし

  【点数表（30符固定）】
    1飜: 子ロン1000 / 子ツモ1000オール / 親ロン2000 / 親ツモ1000オール
    2飜: 子ロン2000 / 子ツモ1000オール / 親ロン3000 / 親ツモ2000オール
    3飜: 子ロン4000 / 子ツモ1000/3000  / 親ロン6000 / 親ツモ3000オール
    4-5飜（満貫）: 子ロン8000 / 子ツモ3000/5000 / 親ロン12000 / 親ツモ6000オール
    6-7飜（跳満）: 子ロン12000 / 子ツモ4000/8000 / 親ロン18000 / 親ツモ9000オール
    8-10飜（倍満）: 子ロン16000 / 子ツモ6000/10000 / 親ロン24000 / 親ツモ12000オール
    11-12飜（三倍満）: 子ロン24000 / 子ツモ8000/16000 / 親ロン36000 / 親ツモ18000オール
    13飜以上（役満）: 子ロン32000 / 子ツモ12000/20000 / 親ロン48000 / 親ツモ24000オール

■ 実装済み役一覧
--------------------------------------------------
  【役満（13飜扱い）】
    天和 / 地和 / 人和 / 流し満貫
    国士無双 / 清老頭 / 字一色 / 緑一色 / 萬子混一色 / 九蓮宝燈
    大三元 / 大四喜 / 小四喜 / 四暗刻 / 四槓子（※特殊条件）
    大車輪（清一色七対子） / 12枚七対子（4枚対子×3）

  【四槓子の特殊条件（GameState.gd / _next_turn）】
    - 1人で4つのカンをすること（他家のカンは含まない）
    - 4つ目のカン後に嶺上牌を引いて1枚捨てる
    - その捨て牌でロンされなければ成立（単騎テンパイ不要）
    - 実装: _count_kantsu(p.naki) >= 4（そのプレイヤーのnakiのみカウント）で
            _suukantsu_pending = player_idx をセット
            → _next_turn(discarder) で pending == discarder なら四槓子役満で和了

  【天和・地和の判定条件（_build_context）】
    天和: dealer かつ junme==1 かつ ツモ かつ p.naki.is_empty() かつ not p.is_riichi
      ・北抜き後も有効（nukita は naki と別管理）
      ・暗槓後は無効（naki に追加されるため）
      ・ダブリー後は無効（is_riichi=true のため）
      ・他家の行動は天和に影響しない（親が最初に行動するため）

    地和: dealer以外 かつ junme==1 かつ ツモ かつ not kyoku_has_meld
      ・北抜き後も有効（_do_kita は kyoku_has_meld をセットしない）
      ・リーチ後も有効（is_riichi は地和条件に含まない）
      ・自分/他家の暗槓・ポン後は無効（kyoku_has_meld=true になる）
      ・他家の北抜きは地和に影響しない

    kyoku_has_meld: bool 変数（GameState）
      _start_kyoku() で false に初期化
      _do_pon() / _do_ankan() で true にセット

  【通常役】
    状況役: リーチ(1) / ダブルリーチ(2) / 一発(1) / 門前ツモ(1)
            嶺上開花(1) / 槍槓(1) / 海底摸月(1) / 河底撈魚(1)
    手役:   タンヤオ(1) / 平和(1) / 一盃口(1) / 二盃口(3)
            七対子(2) / 4枚七対子(4) / 8枚七対子(8)
            対々和(2) / 三暗刻(2) / 三槓子(2) / 小三元(2)
            一気通貫(2/食い1) / 三色同刻(2)
            チャンタ(4/食い3) / 純チャン(6/食い5) / ホンロウトウ(4)
            混一色(3/食い2) / 清一色(6/食い5)
    役牌:   場風(1) / 自風(1) / 白(1) / 發(1) / 中(1)
    平和:   門前・全順子・両面待ち・雀頭が役牌でないこと
            客風の字牌雀頭は平和可、三元牌・場風・自風の雀頭は平和不可

  【ドラ加算】
    通常ドラ（表示牌の次の牌・1枚1飜）
    裏ドラ（リーチ者のみ・1枚1飜）
    赤ドラ（is_red=true の牌・1枚1飜）
    金ドラ（is_gold=true の牌・1枚1飜）
    北抜き（nukita配列の枚数分1飜ずつ）

■ チップシステム（GameState.gd）
--------------------------------------------------
  赤ドラ  : 1枚 = ロン1枚 / ツモ1枚×2人
  金ドラ  : 1枚 = ロン2枚 / ツモ2枚×2人
  赤北    : 1枚 = ロン1枚 / ツモ1枚×2人
  裏ドラ  : 1枚 = ロン1枚 / ツモ1枚×2人
  一発    : ロン1枚 / ツモ1枚×2人
  役満    : ロン10枚 / ツモ5枚×2人
  飛び賞  : 飛ばした側が2枚受け取る（和了者限定）

■ 精算システム（GameState.gd）
--------------------------------------------------
  基準点  : 40000点（開始点数35000とは別）
  ウマ    : 1位+20000 / 2位なし / 3位-20000

  【精算計算方法】
    2位・3位の増減 = 最終点数 - 40000
    1位の増減 = -(2位増減 + 3位増減)（逆算。3者合計がちょうど0）

  【P換算レート】
    点棒P: 1000点 = 100P
    チップP: 1枚 = 500P
    合計P = 点棒P + チップP

■ デバッグモード（game.gd / GameState.gd）
--------------------------------------------------
  F1キー: プレイヤー手牌デバッグパネルの表示/非表示
  F2キー: NPC1（右家）手牌デバッグパネルの表示/非表示
  F3キー: NPC2（上家）手牌デバッグパネルの表示/非表示
  F4キー: 嶺上牌（8枚）デバッグパネルの表示/非表示
  F5キー: NPC手牌の実牌表示/伏せ表示を切り替え
  ゲーム画面の手牌左上にある F1〜F5 ボタンでも同じ操作が可能（初期は非表示・キー操作は常に有効）
  同じキーを再押しするとパネルを閉じる

  【手牌スロット構成（F1/F2/F3・計15スロット）】
    0〜12  : 手牌13枚
    13     : ツモ牌（プレイヤーが今引いている牌）
    14     : 次順ツモ（1周後にプレイヤーが引く牌）
    ※ F1対象時: パネルを開いた初期表示のみ、手牌スロット0〜12を牌ID順にソートする
    ※ NPC対象時: スロット0〜N-1に全手牌配置（ツモ牌もスロットに含む）

  【嶺上牌スロット構成（F4・計8スロット）】
    ①〜⑧: 1回目〜8回目の嶺上ツモ牌
    rinshan.pop_back() で引く順に対応（rinshan配列の末尾から順）
    適用ボタンで GameState.debug_set_rinshan() を呼び出し

  【パレット構成】
    筒子行   : 1〜9筒（通常）
    索子行   : 1〜9索（通常）
    萬/字牌行: 1萬・9萬・東南西北白發中
    特殊牌行 : 赤5筒・赤5索・金8筒・金8索・白ポッチ

  【次順ツモの仕組み（プレイヤー0のみ）】
    ツモ牌が指定されている場合: wall[2] に配置
      （NPC南・NPC西が wall[0]・wall[1] を引いた後が自分の番）
    ツモ牌がない場合: wall[0] に配置

  【適用後】
    プレイヤー0: _check_tsumo_auto() を呼び出し、テンパイ時に即リーチボタン有効化
    NPC: _refresh_npc_areas() を呼び出し手牌表示を更新

  【データ型】
    スロット・パレットはすべて Dictionary 型
    {"id": int, "is_red": bool, "is_gold": bool, "is_haku_pochi": bool}

■ NPC AI（GameState.gd）
--------------------------------------------------
  優先順位:
    1. ツモ和了できればツモ和了宣言
    2. 手牌に北があれば北抜き（門前・副露問わず可能）
       - 北抜き後に嶺上ツモ牌で和了できれば和了宣言
    3. 暗槓できる牌があれば暗槓
    4. 加槓できる牌があれば加槓
    5. 門前・非リーチ状態でスコア1000点以上ならリーチ可能牌で宣言
    6. 上記以外は牌効率打牌

  牌効率打牌:
    - 各打牌候補について打牌後のシャンテン数を計算し、最小シャンテンの牌を切る。
    - 同シャンテンの場合は、打牌後の受け入れ枚数が最大の牌を切る。
    - 受け入れ枚数も同じ場合はランダム選択。ただし金牌・赤牌は同価値候補内で温存する。
    - NPCリーチ宣言牌は、待ち残り枚数 → 最大翻数 → 待ち種類数 → 金牌/赤牌温存 → 安全度の順に評価する。
      同じ牌IDの通常牌と金牌・赤牌がリーチ可能牌に含まれる場合は、通常牌を宣言牌にしてボーナス牌を残す。
    - シャンテン計算は通常手・七対子・国士無双を比較して最小値を使う。
    - リーチ中は従来通りツモ切り。

  ポン判断:
    - 役牌（白・發・中・場風・自風）は積極的にポン
    - 役牌以外はポンしない

  大明槓判断:
    - 役牌を3枚持ちで他家が切った場合は大明槓する

  ロン判断:
    - 他家の捨て牌で通常ロン可能ならNPCが自動ロンする
    - プレイヤー/NPC複数がロン可能な場合は頭ハネ
      （捨てたプレイヤーから次順に近いプレイヤーを優先）
    - プレイヤーが最優先ロンを見逃した場合、次順位のNPCロン候補があれば自動ロンする
    - NPCの捨て牌フリテンは MahjongLogic.is_furiten() で判定

  暗槓判断:
    - ツモ牌で4枚揃ったら暗槓する

  加槓判断:
    - ツモ牌が既存のポンに追加できる場合は加槓する

  チャンカン判断:
    - NPC加槓時：プレイヤーにチャンカン機会を与える（プレイヤーが宣言する）
    - プレイヤー加槓時：NPCに役があれば自動でチャンカン宣言する
    - ポン牌を切った本人はフリテンのためチャンカン不可

  NPCターン遅延: 0.2秒（NPC_THINK_SEC）
  ポン後捨て遅延: 0.1秒

■ キャラクター選択（SaveData.gd / game.gd）

  【鳴き牌表示の共通仕様】
  - 鳴き元牌インデックス: direction = (from_player - holder_idx + 4) % 4
      1(右家から) → 右端 / 2(対面から) → 中央 / 3(左家から) → 左端
      右家NPCがプレイヤーの捨て牌をポンした場合は、鳴いた牌を右家NPC視点の左端（プレイヤー視点の下側）に表示する。
  - 暗槓: 0番と末尾を裏向き（hai_haimen2.webp、STRETCH_KEEP_ASPECT_CENTERED）、中2枚は表向き
  - hai_haimen2.webp は画像の非透明領域を AtlasTexture の region にして、通常牌と同じ枠サイズで表示
  - 鳴き元牌+90°傾き時、position.x に (tile_h - tile_w)/2.0 のオフセットを加算（重なり防止）
  - tiles配列から actual_tile を取得してテクスチャ決定（金牌フラグ保持）
  - ポン/大明槓: 3枚（または4枚）すべて position.y = float(tile_h - tile_w) で下揃え
      鳴き元牌（横向き）の視覚的な底辺が通常牌の底辺と一致する
  - 加槓: ポン3枚と同じ並びで表示し、4枚目（data[3]）を横向き（+90°）で鳴き元牌の真上に重ねる
      加槓牌Y = float(tile_h - 2*tile_w)（ポン回転牌の視覚上辺にぴったり密着、重なりなし）
      pivot_offset = Vector2(tile_w/2, tile_h/2) で中心回転

  【NPC鳴き判定】
  - NPCは他家の捨て牌に対して、三元牌・場風・自風を2枚以上持っている場合は必ずポンする。
  - 同条件で3枚持ちかつ槓可能条件を満たす場合は、大明槓をポンより優先する。

  【プレイヤーのポン・槓選択】
  - ポン時、同種牌を手牌に3枚持ち、赤牌・金牌・白ポッチなどの属性差で選択の余地がある場合は、同種牌だけを選択可能にし、それ以外の牌へ黒マスクを重ねる。
  - ポン選択では、クリックした同種牌を鳴き牌に含め、残り1枚を手牌に残す。
  - 暗槓・加槓で槓可能な牌種が複数ある場合は、槓材だけを選択可能にし、それ以外の牌へ黒マスクを重ねる。
  - 槓選択では、クリックした牌種を暗槓または加槓する。

  【捨て牌のリーチ宣言牌表示】
  - リーチ宣言牌は通常の捨て牌角度に+90°して横向き表示する
  - 横向きで必要になる長辺ぶんの占有幅を確保し、同じ行/ストライプの後続捨て牌だけ追加幅ぶんずらす
  - 追加幅はその行/ストライプ内だけに適用し、折り返し後の行には引き継がない
  - 上家・右家の反転表示では、各プレイヤー視点の進行方向に合わせて追加幅を逆方向へ適用する

■ フリテン判定（GameState.gd）
--------------------------------------------------
  GameState._is_player_furiten(player_idx) で3種を統合判定。
  フリテンはロンにのみ適用。ツモは常に可能。

  ① 捨て牌フリテン（永続）
      自分の河に当たり牌が1枚でも含まれていればフリテン
      → is_furiten() で全待ち牌を自捨て牌と照合（複数待ちに自動波及）

  ② リーチ後見逃しフリテン（永続）
      リーチ中に当たり牌が出てロンをスキップした場合
      → is_riichi_furiten = true。局終了まで解除なし

  ③ 同順フリテン（一時的）
      非リーチで当たり牌を見逃した場合 or 役なしで取れなかった場合
      → is_doujun_furiten = true。次の自分のツモ番で解除

  対局画面のプレイヤーラベルに「※フリテン」を表示

■ 白ポッチ（MahjongLogic.gd / GameState.gd / game.gd）
--------------------------------------------------
  白4枚のうち1枚（is_haku_pochi=true）が白ポッチ。
  牌IDは通常の白（HAKU=45）と同じ。麻雀ロジック上は白として扱われる。

  【通常時】
    他の白と同様に扱う。役牌（白）として和了可能。
    他者が捨てた白ポッチは通常の白としてポン・ロン可能。

  【オールマイティ発動条件】
    「リーチ者が白ポッチをツモった時のみ」発動。
    ・リーチしていない者がツモ → ただの白として扱う
    ・他者が捨てた白ポッチをロン → ただの白として扱う（待ちが白なら和了可）

  【オールマイティ発動時の処理】
    強制和了（プレイヤー確認なし）。
    強制高目どりで待ちの中から最も高い牌を選択して和了扱いにする。

  【高目どり優先順位】
    1位: 飜数が多い方
    2位: チップ枚数が多い方（裏ドラ・一発・役満）
    3位: 字牌 > 萬子 > 筒子 > 索子
    4位: 字牌同士 → 東(41) > 南(42) > 西(43) > 北(44) > 白(45) > 發(46) > 中(47)（ID昇順）
    5位: 数牌同士 → 数字の大きい方

  【result辞書への追加キー】
    haku_pochi_best_tile: int → 採用した当たり牌のID

  【UI表示】
    結果パネル右側に当たり牌の画像を表示（_haku_pochi_img: TextureRect）
    ツモ牌欄には白ポッチの画像のまま表示（役計算用の置き換えはUI表示後に復元）
    ※ _build_win_result() 後に p.hand を白ポッチ(id=45, is_haku_pochi=true)へ復元

■ 北牌の和了制限（MahjongLogic.gd）
--------------------------------------------------
  北（id=44）は三麻では「抜き牌」として機能するため、手牌に残っている場合、
  通常の和了形には使用できない。

  北が手牌に含まれる場合、以下の役満形のみ和了可：
    ① 国士無双 → 北は幺九牌として計上される
    ② 字一色   → 全牌が字牌の場合、北も通常字牌として使用可
    ③ 大四喜   → 東南西北 の全4風が刻子以上の手
    ④ 小四喜   → 3風が刻子以上 + 1風が対子（北が刻子または対子）

  実装箇所:
    is_complete_hand(): 北入り手は _is_valid_north_hand() で役満形のみ true
    _is_valid_north_hand(): 上記①〜④を順にチェック
    _is_north_in_siisou(): 大四喜・小四喜の風牌構成チェック
    check_yaku(): 役満チェック後に北入り手なら [] を返す（二重防御）

  ※ 北抜きをせずに手牌に北を温存していても、役満以外では和了不可

■ 2026/05/28 追加仕様：打牌シミュレーター・牌譜機能
--------------------------------------------------
  【シミュレーター（SimulatorScreen.gd）】
  - メニュー右パネル「シミュレーター」ボタン（y=780）から起動。
  - レイアウト：game.gd の背景・スタイル（bg_takujou.webp）を流用。
    左パネル（x=0..700）: 解析結果（全候補＋有効牌を常に展開表示）
    中央パネル（x=700..1300）: パレットタブ・グリッド + 手牌スロット14枚 + アクションボタン行
    右パネル（x=1300..1920）: 残り山SpinBox + 見えている牌（折りたたみ）
  - アクションボタン行：[▶ 解析] [クリア] [🔍 検討]（検討ボタンは見えている牌パネルのトグル）
  - キャラクター（クマ立ち絵）は表示しない。
  - 牌譜ビューアの「検討」から起動した場合は static var（initial_hand / initial_dead_tiles / initial_total_wall）で状態を受け取る。

  【SanmaAnalyzer（SanmaAnalyzer.gd）】
  - evaluate_discards(hand, total_wall=61, dead_tiles={}, meld_count=0) → Array
      meld_count: 副露済み面子数（game.gd からは GameState.players[0].naki.size() を渡す）
  - dead_tiles 構造: {"counts":{game_id:枚数}, "red":{game_id:赤枚数}, "gold":{game_id:金枚数}}
  - 返り値の各辞書キー:
      tile_id / tile_name / shanten / shanten_text / effective_tiles
      effective_count        : 有効牌生枚数（補正なし）
      effective_count_expected: 有効牌期待値枚数（× 山確率 total_wall/(total_wall+34)）
      tenpai_rate            : シャンテン依存（下記参照）
      agari_rate             : シャンテン依存（下記参照）
      tile_breakdown         : 有効牌ごとの内訳 {game_id: {wall_count, expected(float均等割り), tenpai_rate, agari_rate}}
  - calc_shanten(counts, meld_count=0):
      meld_count > 0 のとき七対子向聴を除外し、一般手のみで計算する。
      キャッシュキーは counts を2分割した Vector3i と meld_count で構成し、配列文字列化を避ける。
  - _shanten_regular(counts, meld_count=0):
      _best の初期値を 8 - meld_count * 2 に設定（副露済み面子は完成扱い）。
      _search() の mentsu 開始値を meld_count に設定。
      _search() 内では counts・探索位置・面子数・搭子数・雀頭有無の部分状態をメモ化し、同一状態の再帰探索を省略する。
  - _calc_effective_tiles(counts, shanten, dead_tiles, meld_count=0):
      wall_count = DECK_SIZE - 手牌 - 死牌（生の枚数、山確率補正なし）
      shanten==0 のときは、残り0枚の待ちも UI 表示用に effective_tiles[game_id] = 0 として残す。
  - _calc_agari_rate(e, w): 累乗計算ヘルパー（shanten==0/1/2/3で共通）
      turns = ceil(w/3)、p_per_draw = minf(e / w, 1.0)
      return minf(1.0 - pow(1.0 - p_per_draw, turns), 1.0)
  - _calc_rates() シャンテン別の意味:
      evaluate_discards() の2パス目から meld_count を引き継ぎ、詳細確率内の向聴数・有効牌再計算でも副露済み面子を完成扱いにする。
      shanten==0: tenpai_rate=1.0, agari_rate=ツモ和了率（clamp 1.0）
      shanten==1: tenpai_rate=テンパイ到達率, agari_rate=和了期待値（両方 clamp 1.0）
                  tile_breakdown = 各有効牌の{tenpai_rate=引く確率, agari_rate=テンパイ後和了率,
                                              expected=均等割り期待枚数(float)}
      shanten==2: tenpai_rate=1シャンテン到達率(clamp), agari_rate=テンパイ到達率(clamp)
                  tile_breakdown = 各有効牌の{tenpai_rate=引く確率, agari_rate=-1.0,
                                              expected=均等割り期待枚数(float)}
      shanten>=3: tenpai_rate=1シャンテン到達率, agari_rate=-1.0, tile_breakdown={}
  - effective_count_expected: float（生の枚数 × 山確率）
  - 表示ラベル（shanten別）フォントサイズ:
      メイン情報（牌名・向聴テキスト・確率）: font 35
      有効牌枚数「有効N枚（期待X.X枚）」: font 28
      breakdown各行（牌画像＋テキスト）: font 24
  - 表示ラベル（shanten別）テキスト:
      ==0 : 「和了率 XX.XX%」のみ（font 35）
      ==1 : 黄色表示は「和了率 XX.XX%」のみ（青文字の次巡聴牌確率と重複するため黄色のテンパイ率は表示しない）
      ==2 : 「1シャンテン率 XX.XX%」（font 35）「テンパイ率 XX.XX%」（font 28）
      >=3 : 「1シャンテン率 XX.XX%」のみ（font 35）
      tenpai_r<0: 「—」
  - breakdown行: 牌画像(44×60) ＋「×N（期待X.X枚）」＋「引く確率X.XX%」＋「テンパイ率/繰り上げ率X.XX%」
      行高 = 90 + breakdown行数×68 + 8
  - 有効牌グリッド表示は廃止（breakdown行に統合）

■ 2026/05/18 追加仕様：槓制限・リーチ後暗槓・暗槓槍槓
--------------------------------------------------
  【全槓共通の不可条件】
  - ツモ山の残り枚数が2枚未満（0枚または1枚）の場合、暗槓・大明槓・加槓は不可。
  - 既にその局で4回槓が成立している場合、5回目の槓は不可。

  【リーチ後の暗槓】
  - リーチ後に暗槓候補になるのは、リーチ後にツモってきた牌のみ。
  - リーチ時点で既に手牌内に4枚あった牌は、リーチ後に暗槓できない。
  - リーチ後にツモった牌が、既存手牌内の同一牌3枚と合わせて4枚になる場合だけ候補とする。
  - 暗槓後の待ち牌集合がリーチ宣言時に保存した待ち牌集合と完全一致する場合のみ暗槓可。
  - 暗槓によって待ち牌が変化する場合は不可。

  【暗槓に対する槍槓】
  - 通常、槍槓は加槓のみ対象とし、暗槓に対する槍槓は不可。
  - 例外として、国士無双で和了できる場合に限り、暗槓に対する槍槓を認める。
  - 暗槓に対する槍槓機会をスキップした場合は、暗槓処理を続行する。

  - 1回の捨て牌処理で、対象プレイヤーの手牌は必ず1枚だけ減る。
  - 捨て牌処理で不正な手牌インデックスが指定された場合は、処理を中断し `Invariant violation` をGodotログへ出す。
  - 捨て牌直後は一旦 `ACTION_WAIT` に入り、ロン・ポン・カン等の割り込み判定が完了するまで追加の自動打牌を通さない。
  - 最後に捨てたプレイヤーIDと最後の捨て牌は常に整合している必要がある。不整合時は `Invariant violation` をGodotログへ出す。
  - 各プレイヤーの手牌枚数、および手牌+副露牌枚数が明らかに破綻した場合は `Invariant violation` をGodotログへ出す。

  【リーチ中自動ツモ切り】
  - プレイヤーのリーチ中自動ツモ切りタイマーは同時に1本だけ予約する。
  - 捨て牌・ツモ和了・ロン機会発生時は、予約中の自動ツモ切り状態を解除する。
  - NPCがプレイヤーの捨て牌に対してポン・大明槓を予約した時点で `ACTION_WAIT` に入り、NPC鳴き処理が実行されるまで次の自動ツモ切りが割り込まないようにする。

  【鳴かれた捨て牌の管理】
  - ポン・大明槓で黒くする捨て牌は `discards[-1]` 決め打ちにしない。
  - 鳴いた牌と同じ牌ID・赤/金/白ポッチ属性を持つ未取得捨て牌を、対象プレイヤーの河の後方から検索して `is_taken` を立てる。
  - 鳴き対象牌が河に見つからない場合は `Invariant violation` をGodotログへ出す。

  【オープン立直放銃不可】
  - NPCはリーチ中でない限り、オープン立直者の待ち牌を通常打牌・性格別打牌・ベタオリ打牌・リーチ宣言牌として選ばない。
  - 切れる牌が全てオープン立直者の待ち牌だった場合のみ、例外Aとして当たり牌を切れる。
  - NPC自身がリーチ中で強制ツモ切りになる場合は、例外Bとして当たり牌を切れる。

■ 2026/05/30 追加仕様：打牌アシスト機能（game.gd）
--------------------------------------------------
  【アシストボタン・パネル概要】
  - 旧アシストボタン位置に「アシストモード」ラベルと、打牌アシストON/OFF用のiPhone風トグルを表示する。
  - トグルは size(198,87)・position(1682,620)、ON/OFF の丸を大きく表示する。
  - OFF時はトグルをグレー表示にし、★・左パネルを非表示にする。
  - assist_mode に応じて、プレイヤー打牌可能時に自動解析する。
      0=OFF: 表示しない
      1=★だけ表示: 最善打牌の上に★のみ表示
      2=★＋左パネル常時表示: ★と左パネル上位3候補を常時表示
  - ロン/ポン/カン/リーチ選択中、リーチ中、プレイヤー打牌不能時は非表示。

  【事前データキャッシュ】
  - _on_tile_drawn でプレイヤーのツモ時に dead_tiles（_build_assist_dead_tiles()）と
    wall サイズを事前収集して _assist_cached_dead_tiles / _assist_cached_total_wall に保存。
  - _assist_cache_ready フラグが true の場合はキャッシュ値を使用し、false の場合は
    ボタン押下時にその場で収集する。
  - _hide_assist() 時に _assist_cache_ready を false にリセット。

  【解析実行フロー】
  - _on_assist_pressed():
    1. assist_enabled をON/OFFする。
    2. OFF時は★・左パネルを非表示にする。
    3. ON時は現在の assist_mode に応じて _refresh_auto_assist() を実行する。
  - _refresh_auto_assist():
    1. SanmaAnalyzer.evaluate_discards() で打牌候補を計算。
    2. _apply_tiebreak_priority() でソート。
    3. assist_mode==1 なら _place_assist_star()、assist_mode==2 なら _show_assist() で結果表示。

  【_show_assist() 行レイアウト（1件あたり）】
  - パネル幅: 444 / 行高（custom_minimum_size）: 270 / 行間隔（py加算）: 278
  - 牌画像: position(6, 6)・size(66, 90)
  - ★マーク（ベストのみ）: position(76, 6)・font 36pt
  - 牌名ラベル: position(100, 6)・font 32pt
  - 向聴テキスト: position(100, 44)・font 26pt
  - 有効枚数「有効N枚（期待X.X枚）」: position(6, 100)・font 40pt
  - 次巡確率「次巡〇〇確率 XX.XX%」: position(6, 154)・font 40pt
  - 有効牌画像: size(33, 45)・y=210 から横並び（ex += 50、ex > 390 で打ち切り）
  - 有効牌枚数ラベル: font 22pt・position(ex+35, 222)

  【_next_turn_label(shanten) — シャンテン別ラベル】
  - 0 → 「次巡和了確率」
  - 1 → 「次巡聴牌確率」
  - 2 → 「次巡一向聴確率」
  - それ以外 → 「次巡繰り上げ確率」
  同関数を SimulatorScreen.gd / GameLogViewerScreen.gd にも実装。

  【★表示（_place_assist_star）】
  - _rebuild_player_hand と同じ sort_custom + _tile_sort_key ロジックで display_indices を再構築。
  - ★を該当ボタンの子ノードとして追加（position: Vector2(btn.size.x/2-20, -44)・font 40pt・z_index 10）。
  - _show_assist_loading / _show_assist / _hide_assist の3箇所で
    is_instance_valid() チェック後に queue_free することでクラッシュを防ぐ。

  【解析中表示（SimulatorScreen / GameLogViewerScreen）】
  - ▶解析ボタン（SimulatorScreen）: _no_result_lbl に「解析中...」を表示後、
    call_deferred("_run_analysis_deferred") で計算を遅延実行。
  - 🔍検討ボタン（GameLogViewerScreen）: _result_list をクリアして「解析中...」ラベルを追加後、
    call_deferred("_run_sim_analysis_deferred") で計算を遅延実行。
