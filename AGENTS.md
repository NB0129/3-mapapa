# 3-mapapa 開発ルール

## 基本方針
- 三麻ローカルルール前提で扱う
- 不明な仕様を推測で実装しない
- 大規模リファクタリングを避ける
- 既存構造をなるべく維持する

## 作業ルール
- 修正前に関連ファイルを調査する
- 一度に複数の大規模変更をしない
- 修正後は変更ファイルと内容を報告する
- Godotプロジェクトを壊す可能性がある変更は事前に説明する

## アセット運用
- PNG画像をWebPへ変換した場合、実行時に使うWebPと `.webp.import` を残し、変換元PNGと `.png.import` は削除する

## ドキュメント
- 実装・修正後は仕様書を更新する
- 実装・修正後は更新履歴を更新する

## Git運用
- 修正後は git add . を行う
- 修正後は git commit を行う
- commitメッセージは具体的に書く
- 修正項目ごとにcommitを分ける

## UI関連
- UI座標や牌表示方向は既存ルールを優先する
- プレイヤー視点とNPC視点の違いを意識する
